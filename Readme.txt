Задание 1:
Менеджер продаж (далее МП) ППР, в процессе коммерческих переговоров, сохраняет все свои
действия в CRM системе. Некоторые действия – это составляющие одного бизнес-процесса/этапа
Задача - рассчитайте непрерывную длительность этапа по каждой компании.
Описание полей таблиц
«TASK_1.SALES_STAGE_DICT»– справочник названий действий и бизнес-процессов:
- HIGH SALES STAGE – общее название бизнес-процесса
- DETAIL SALES STAGE – действия сотрудника с компанией
«TASK_1.SALES_STAGE_LOG»– таблица с логами активностей в CRM :
- ID – идентификатор компании в CRM
- DATE – дата изменения значения в CRM
- NEW_VALUE – новое значение
- OLD_VALUE – старое значение

Решение:
	В результате выполнения работы исходные таблицы из Excel были импортированы в Python, где была реализована основная логика обработки данных,
после чего, с помощью SQLite сформирована база данных и в командном окне выведены три таблицы: 
	 - Таблица 1 "SALES_STAGE_DICT" - исходная таблица из условия задания,
	 - Таблица 2 "SALES_STAGE_LOG" - исхдная таблица с доп. полями начала и конца выполнеиня этапа 
	 - Таблица 3 - результат выполнения технического задания, где выведена длительность выполенеия каждого этапа по каждой компании.

	В архиве находятся файл 'exfile.xclx' с исходными таблицами в формате excel, основной файл 'logs.py', в результате исполнения
которого выводятся итоговые таблицы, 'logs.db' - база данных (также три фалйа jpg - результаты вывода таблиц).
	Для получения результата необходимо открыть файл 'logs.py' и запустить программу
